Design for System Performance & Scalability

ARCHITECT FOR FAILURE

ARCHITECT FOR FAILURE

Failure is inevitable, architect for it. Ensure that each application component can fail gracefully and recover.

ENSURE REDUNDANCY

ENSURE REDUNDANCY

Redundant hardware, software and geographic locations are key to always available systems. Applications have to be aware and play well.

BAKED-IN SECURITY

BAKED-IN SECURITY

Security baked into each and component. Assume a trust no one approach with zero tolerance.

TIERED PERFORMANCE

TIERED PERFORMANCE

Security and performance come with costs. Architecture should manage trade-offs between needs and costs.

DYNAMIC ELASTICITY

DYNAMIC ELASTICITY

Architecture should incorporate automated provisioning, deployment, scaling and failure recovery. Efficient operations and improves manageability.

MANAGE LATENCY

MANAGE LATENCY

Co-locate components and data wherever relevant. Eliminates costly network and IO related latency.

FOCUS ON DATA FLOWS

FOCUS ON DATA FLOWS

Data architecture is key to high performance systems. Architecture should ensure that access, durability, consistency, availability, volume of data are addressed.

PARTITION AND PARALLELIZE

PARTITION AND PARALLELIZE

Wherever possible architect to partition and parallelize. This approach allows for efficient, high performance systems with low latency.

LOOSELY COUPLE

LOOSELY COUPLE

Components should be loosely coupled and interface dependencies should be at a minimum.
